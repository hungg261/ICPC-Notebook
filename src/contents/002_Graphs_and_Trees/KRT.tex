% Kruskal Reconstruction Tree (KRT)

A tree structure built on top of DSU by processing edges in increasing order of weight. It allows path max/min edge queries between any two vertices to be reduced to an LCA query in $O(1)$ after preprocessing.

\begin{lstlisting}
struct KRT {
    int n, node_cnt;
    vector<int> parent, sz, node_weight;
    vector<vector<int>> adj;

    KRT(int n) : n(n), node_cnt(n), parent(2 * n + 1), sz(2 * n + 1, 1), node_weight(2 * n + 1, 0), adj(2 * n + 1) {
        iota(parent.begin(), parent.end(), 0);
    }

    int find_set(int v) {
        if (v == parent[v]) return v;
        return parent[v] = find_set(parent[v]);
    }

    void build(vector<tuple<int, int, int>>& edges) {
        // Sort edges by weight for MST order
        sort(edges.begin(), edges.end());

        for (auto [w, u, v] : edges) {
            int root_u = find_set(u);
            int root_v = find_set(v);

            if (root_u != root_v) {
                node_cnt++;
                node_weight[node_cnt] = w;

                parent[root_u] = node_cnt;
                parent[root_v] = node_cnt;
                
                adj[node_cnt].push_back(root_u);
                adj[node_cnt].push_back(root_v);
            }
        }
    }
};
\end{lstlisting}

Pass graph edges as \lstinline|{weight, u, v}| array to \lstinline|build(edges)|. The resulting reconstruction tree contains at most $2N - 1$ nodes where original vertices are leaves ($1 \le i \le N$). The maximum edge weight on the path between $U$ and $V$ in the original graph is exactly equal to \lstinline|node_weight[LCA(u, v)]|.
